# Embedding-Friendly Memory Writing

Guidelines for writing memory files that work well with vector retrieval.

## Rules

- One topic per file, one subtopic per section
- Lead each section with the concept and key fact
- Write proper nouns exactly and consistently
- Prefer prose for conceptual material, bullets for true lists
- Keep Related sections small and navigational
- Update files in place instead of appending stale history

## Anti-patterns

- Do not stuff keywords, synonyms, or multi-language expansions
- Do not add reformulated concepts sections
- If retrieval feels weak, expand the query at search time instead of diluting documents
